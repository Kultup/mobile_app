# Технічне завдання на дизайн мобільного додатку

## 1. Загальні принципи дизайну

### 1.1. Стиль
- **Сучасний мінімалістичний дизайн** з акцентом на контент
- **Material Design** для Android та **Human Interface Guidelines** для iOS
- **Адаптивність** під різні розміри екранів та вирізи
- **Зручність використання** - інтуїтивна навігація, зрозумілі дії

### 1.2. Цільова аудиторія
- Працівники ресторанної мережі (офіціанти, кухарі, менеджери)
- Вік: 18-55 років
- Різний рівень технічної підготовки
- Використання в робочий час та після роботи

### 1.3. Основні принципи UX
- **Простота** - мінімум кроків для виконання дій
- **Швидкість** - швидкий доступ до основних функцій
- **Мотивація** - візуальне підкреслення досягнень та прогресу
- **Офлайн-доступність** - індикація доступності контенту без інтернету

## 3. Типографіка

### 3.1. Шрифти

**iOS:**
- Основний: **SF Pro Display** (системний)
- Альтернатива: **Inter** або **Roboto**

**Android:**
- Основний: **Roboto** (системний)
- Альтернатива: **Inter** або **SF Pro Display**

### 3.2. Розміри шрифтів

```
H1 - Заголовок екрану: 28px (Bold)
H2 - Заголовок секції: 24px (Bold)
H3 - Підзаголовок: 20px (SemiBold)
Body Large - Основний текст: 16px (Regular)
Body - Звичайний текст: 14px (Regular)
Body Small - Дрібний текст: 12px (Regular)
Caption - Підписи: 11px (Regular)
```

### 3.3. Висота рядків
```
H1: 36px (line-height: 1.3)
H2: 32px (line-height: 1.3)
H3: 28px (line-height: 1.4)
Body: 20px (line-height: 1.4)
Body Small: 16px (line-height: 1.4)
```

### 3.4. Відступи між елементами
```
Між заголовками та текстом: 8px
Між параграфами: 16px
Між секціями: 24px
Між великими блоками: 32px
```

## 7. Анімації та переходи

### 7.1. Переходи між екранами
- **Тип:** Slide (горизонтальний)
- **Тривалість:** 300ms
- **Easing:** ease-in-out

### 7.2. Анімації появлення
- **Fade In:** 200ms
- **Slide Up:** 300ms з fade
- **Scale:** для модальних вікон

### 7.3. Інтерактивні анімації
- **Натискання кнопки:** scale 0.95 на 100ms
- **Swipe:** для карток та списків
- **Pull to Refresh:** стандартна анімація

### 7.4. Спеціальні анімації
- **Отримання ачівки:** конфеті ефект, scale + rotate
- **Правильна відповідь:** зелена хвиля, checkmark анімація
- **Неправильна відповідь:** червоний shake, X анімація
- **Покупка товару:** монети/балі анімація

## 8. Стани інтерфейсу

### 8.1. Loading (Завантаження)
```
Skeleton screens замість спінерів:
- Сіра картка з shimmer ефектом
- Анімація: 1.5s, ease-in-out, infinite
```

### 8.2. Empty State (Порожній стан)
```
Ілюстрація + текст:
- Велика іконка/ілюстрація
- Заголовок: 20px, SemiBold
- Опис: 14px, Regular, #666666
- Кнопка дії (якщо потрібно)
```

### 8.3. Error State (Помилка)
```
Іконка + текст + кнопка:
- Червона іконка помилки
- Заголовок: "Щось пішло не так"
- Опис помилки
- Кнопка "Спробувати ще раз"
```

### 8.4. Success State (Успіх)

**iOS:**
- Safe area top: мінімум 44px
- Safe area bottom: мінімум 34px (для home indicator)

**Android:**
- Враховувати різні розміри статус-бару
- Враховувати gesture navigation vs buttons

- **Портретна:** основна орієнтація
